# Probabilistic Earley Parse

**Author:** Pratishtha Saxena

## Overview
This repository contains a Python implementation of a Probabilistic Earley Parse. The parse takes a Probabilistic Context-Free Grammar (PCFG) and an input sentence, applying the Earley algorithm to find all valid syntactic derivations. 

It utilizes Viterbi decoding (minimizing negative log probabilities, or weights) to resolve structural ambiguities and identify the most likely parse tree.

## Features
* **Viterbi Decoding:** Computes the probability of each derivation and guarantees the selection of the highest-probability parse.
* **LISP-Style Tree Formatting:** Reconstructs the best parse tree using detailed backpointers and outputs it in a deeply nested, mathematically aligned LISP format.
* **Span Extraction:** Explicitly tracks and outputs the `[NonTerminal, start_index, end_index]` constituency spans for every node in the winning tree.
* **Batch Processing:** Can automatically detect and run all grammar/sentence pairs in a directory if no specific files are provided.
* **Hash-Based Agenda:** Ensures $O(1)$ duplicate detection for chart items, maintaining optimal $O(n^3)$ time and $O(n^2)$ space complexity.

## Requirements
This script relies exclusively on Python standard libraries. No external dependencies are required.
* Python 3.x
* `sys`, `os`, `math`, `collections`

## Usage

### 1. Single File Mode
To parse a specific sentence file using a specific grammar file, pass them as command-line arguments:

python parse.py grammar_file.gr sentence_file.sen

### 2. Batch Mode

If you run the script without any arguments, it will automatically scan the current directory for all .gr files and match them with corresponding .sen files (e.g., time.gr with time.sen):

python parser.py

---

## Input File Formats

### Grammar File (.gr)

- Space-separated format  
- First column → rule probability  
- Second column → Left-Hand Side (LHS)  
- Remaining columns → Right-Hand Side (RHS)  

Example:
1.0 S NP VP
0.5 V flies
...

---

### Sentence File (.sen)

- Each line contains one sentence  
- Words are separated by whitespace  
- Blank lines are ignored  

Example:
time flies like an arrow

---

## Output Format

If the sentence is successfully parsed, the parser outputs the following for each valid parse tree:

1. The parse tree (LISP-style format)  
2. The probability of that parse  
3. The list of constituency spans  

---

### Example Output

Parse 1:
(ROOT (S (NP (N time))
         (VP (V flies)
             (ADVP (ADV like)
                   (NP (D an)
                       (N arrow))))))
Probability: 4.480000e-03
Spans:
[N, 0, 1]
[NP, 0, 1]
...

---

### Multiple Parses

- If multiple valid parses exist (due to ambiguity), all parse trees are enumerated
- Each parse is printed with its corresponding probability  
- This satisfies the requirement to compute probability of each parse tree

---

### Failure Case

If the sentence cannot be generated by the grammar:

NONE

---

## Additional Notes

- Probabilities are computed using log-space (negative log probabilities) internally for numerical stability  
- Final probabilities are converted back to standard form before printing  
- The parser retains Viterbi efficiency while also supporting full parse enumeration  
- Constituency spans are extracted for each parse tree using recursive traversal and printed in sorted order
